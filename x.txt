# Annotation-Driven Search Query JSON Schema

## Summary

Replace the hand-built JSON nodes in SearchQuerySchemaService with VicTools schema generation driven by annotations on the source model classes under @src/main/java/com/monk3/model. The /queries/schema endpoint stays the same, but the schema content becomes derived from Java annotations instead of duplicated manual schema code.

Use MicroProfile OpenAPI @Schema annotations as the schema metadata source, plus existing Jackson and Jakarta validation annotations. The generated schema should preserve the richer checked-in search-query-dsl.schema.json intent and also expose validation-derived constraints such as minLength from @NotBlank.

## Key Changes

- Annotate the model records/interfaces with MicroProfile @Schema metadata:
    - Root title, description, and  on SearchQueryRequest.
    - Property descriptions on record components.
    - constValue = text / range on TextQuery.type and RangeQuery.type.
    - oneOf on QueryPayload for TextQuery and RangeQuery.
    - if / then / else schema annotations on QueryNode using hidden nested marker classes for the field ==  data-shape rule.
    - anyOf / not schema annotations on RangeQuery using hidden marker classes for “at least one bound” and conflicting lower/upper bounds.
- Add a small VicTools module in com.monk3.schema that reads MicroProfile @Schema and @SchemaProperty annotations:
    - Map title, description, comment, constValue, oneOf, anyOf, allOf, not, ifSchema, thenSchema, elseSchema, requiredProperties, and properties.
    - Support additionalProperties = Schema.False.class.
    - Support local @Schema.extensions keys for JSON Schema details MicroProfile cannot express directly: x-json-schema-id, x-json-schema-type, and item-description extensions for array items.
- Simplify SearchQuerySchemaService:
    - Configure VicTools with SchemaVersion.DRAFT_2020_12, JacksonModule for Jackson naming/descriptions but with subtype schema transforms disabled,
      JakartaValidationModule, the new MicroProfile schema module, SCHEMA_VERSION_INDICATOR, FORBIDDEN_ADDITIONAL_PROPERTIES_BY_DEFAULT, and stable generation.
    - Make generateSchema() return schemaGenerator.generateSchema(SearchQueryRequest.class) without rebuilding properties or  manually.
- Add custom schema handling for JSON-value wrapper models:
    - RangeBound generates inline type: [number, string].
    - BooleanQueryData generates inline array -> array -> QueryNode from its @JsonValue shape.

## Public Interfaces

- No REST API path, request, or response contract changes.
- The generated schema will include additional annotation-derived validation keywords, especially minLength and minItems, where the Java model already declares @NotBlank or @NotEmpty.
